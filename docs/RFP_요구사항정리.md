# 프로젝트 요구사항 정리 (RFP)

## Must Have (반드시 포함 - 개발 의뢰 문서 필수 요구사항)

### ① 제공 JSON 데이터 연동
- 5개 권역 중 1곳 선정하여 프론트/백엔드 연동
- 직접적인 공공 API 호출 없이 **제공된 JSON 파일만 활용**

### ② 익명 커뮤니티 CRUD
- 회원가입/로그인 없는 익명 게시판
- 비밀번호 **평문 저장**, 수정/삭제 시 일치 여부로 권한 확인
- 게시글 목록, 상세 조회, 작성, 수정, 삭제 **REST API 및 화면** 구현
- 데이터는 반드시 **SQLite DB**에 저장

### ③ OpenAI 기반 챗봇
- `POST /api/chat` 엔드포인트 구축 (OpenAI API 연동)
- 제공 JSON 데이터를 바탕으로 관광지/축제 등 자연어 질의응답
- 대화 히스토리 유지
- 모바일 대응
- 플로팅 UI 위젯 형태로 구현

### ④ 필수 아키텍처 & 배포
- **Vue.js 3 (SPA)** + **FastAPI** + **SQLite** 스키마 설계
- 민감 정보(API 키 등) `.env` 관리, **Git 커밋 절대 금지** (`.gitignore` 처리)
- 배포
  - 프론트엔드: **Netlify**
  - 백엔드: **Render**
- 최종 산출물에 **배포 URL 포함**

## Should / Could Have (선택 - 팀이 선정한 기능으로 교체 가능)

### ① 데이터 시각화 대시보드 (선택 기능 1순위)
- Chart.js 또는 D3.js를 활용한 권역별 게시글 통계 시각화
- 카테고리별 게시글 비율, 인기 관광지/지역 통계 차트 제공
